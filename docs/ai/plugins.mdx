<Callout variant="info" icon="🧪">

Storybook's AI capabilities are currently in [preview](../releases/features.mdx#preview). The experience may change in future releases. We welcome feedback and contributions to help improve this feature.

</Callout>

You can use Storybook's plugin in Claude Code, Claude Desktop, and Codex to connect agents to your Storybook. Agents can then use the plugin's toolsets to generate UI, run tests, and preview their work in your Storybook. If you're using an Agentic Development Environment (ADE), the agent can automatically open relevant stories in the ADE preview, so you can inspect the components and their code.

[TK - image - Story open in ADE preview]
![An ADE working on a UI task with a story open in the preview](../_assets/ai/ade-preview-story.png)

## Requirements

- [Claude Code](https://claude.ai/code), [Claude Desktop](https://claude.ai/desktop), or [Codex](https://openai.com/codex/).

## Installation

<Callout variant="info" icon="ℹ️">

Because the plugins are in preview, they have not yet been added to Claude or Codex's marketplace. These instructions guide you to add the Storybook marketplace to your harness and install the plugin from there.

</Callout>

<details>
<summary><h3 id="claude-code"><strong>Claude Code</strong></h3></summary>

1. Run this command to add the Storybook marketplace to Claude Code:

   ```bash
   claude plugin marketplace add storybookjs/mcp@main --scope user
   ```

2. Then install the plugin:

   ```bash
   claude plugin install storybook@storybook --scope user
   ```

3. Confirm the plugin is available:

   ```bash
   claude plugin list
   ```

You're all set!

</details>

<details>
<summary><h3 id="claude-desktop"><strong>Claude Desktop</strong></h3></summary>

1. Run this command to add the Storybook marketplace to Claude Desktop:

   ```bash
   /plugin marketplace add storybookjs/mcp@main --scope user
   ```

2. Then install the plugin:

   ```bash
   /plugin install storybook@storybook --scope user
   ```

3. Confirm the plugin is available:

   ```bash
   /plugin
   ```

   Then go to the Installed tab to confirm the plugin is listed.

You're all set!

</details>

<details>
<summary><h3 id="codex"><strong>Codex</strong></h3></summary>

1. Run this command to add the Storybook marketplace to Codex:

   ```bash
   codex plugin marketplace add storybookjs/mcp --ref main
   ```

2. Then install the plugin:

   ```bash
   codex plugin add storybook@storybook
   ```

3. Confirm the plugin is available:

   ```bash
   codex plugin list --marketplace storybook
   ```

You're all set!

</details>

### Update a plugin

Until the plugins are available in the official marketplaces, you can update a plugin by removing it and following the installation instructions again.

To remove a plugin:

<details>
<summary>Claude Code</summary>

1. Run this command to remove the Storybook marketplace, which will also uninstall the plugin:

   ```bash
   claude plugin marketplace remove storybook --scope user
   ```

2. Follow the [installation instructions above](#claude-code) to re-add the marketplace and install the plugin again.

</details>

<details>
<summary>Claude Desktop</summary>

1. Run `/plugin` and go to the Marketplaces tab

2. Remove the "storybook" marketplace

3. Follow the [installation instructions above](#claude-desktop) to re-add the marketplace and install the plugin again.

</details>

<details>
<summary>Codex</summary>

1. Run this command to remove the plugin from Codex:

   ```bash
   codex plugin remove storybook@storybook
   ```

2. Then remove the Storybook marketplace:

   ```bash
   codex plugin marketplace remove storybook
   ```

3. Follow the [installation instructions above](#codex) to re-add the marketplace and install the plugin again.

</details>

## Usage

The plugin includes instructions to help the agent understand how and when to use the [skills](#skills) and [tools](#tools) available to it. As your agent works on UI tasks, it can use the plugin to generate stories, run tests, and preview its work in your Storybook. You can also explicitly call the plugin's skills in prompts (e.g. `/stories`) to have the agent perform specific actions.

If you're using an Agentic Development Environment (ADE) like Claude Desktop or Codex, the agent will use the plugin to automatically open relevant stories or an [agentic review summary](./agentic-review.mdx) in the ADE preview, so you can review the agent's work.

## Skills

These skills are available to agents that have the Storybook plugin installed. They can be referenced in prompts (e.g. `/stories`) or the agent can use them indirectly while working on a task.

### `init`

Initializes Storybook in your project, if it isn't already set up. This is the same as running [`npx storybook init`](../get-started/install.mdx) in your project.

### `setup`

Writes basic stories for your project.

### `stories`

TK

### `upgrade`

Upgrades your Storybook to the latest version. This is the same as running [`npx storybook upgrade`](../releases/upgrading.mdx) in your project.

## Tools

All of [Storybook MCP server's tools](./mcp/overview.mdx#toolsets) are available to agents that have the plugin installed.
