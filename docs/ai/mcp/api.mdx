## `@storybook/addon-mcp` options

The MCP server addon accepts the following options to configure the tools provided by the MCP server. You can provide these options when registering the addon in your `main.js|ts` file:

<CodeSnippets path="addon-mcp-options.md" />

### `toolsets`

Type:

```ts
{
  dev?: boolean;
  docs?: boolean;
  test?: boolean;
}
```

Default: 

```ts
{
  dev: true,
  docs: true,
  test: true,
}
```

Configuration object to toggle which toolsets are enabled in the MCP server. By default, all toolsets are enabled.

#### `dev`

Type: `boolean`

Default: `true`

The development toolset includes the [`get-storybook-story-instructions`](#get-storybook-story-instructions) and [`preview-stories`](#preview-stories) tools.

#### `docs`

Type: `boolean`

Default: `true`

The docs toolset includes the [`get-documentation`](#get-documentation) and [`list-all-documentation`](#list-all-documentation) tools.

#### `test`

Type: `boolean`

Default: `true`

The testing toolset includes the [`run-story-tests`](#run-story-tests) tool.

## MCP server tools

### `get-documentation`

Type: 

```ts
async (input: { id: string; storybookId?: string }) => {
  content: [
    {
      type: 'text';
      text: string;
    }
  ],
  isError?: boolean;
}
```

Returns the detailed documentation for a specific component, including its props, usage examples, and any additional documentation you've provided.

Part of the [docs toolset](#docs).

### `get-storybook-story-instructions`

Type: `TK`

Returns instructions to author useful stories, including which props to capture and how to write interaction tests for the story (if applicable).

Part of the [development toolset](#dev).

### `list-all-documentation`

```ts
async (input: { withStoryIds?: boolean }) => {
  content: [
    {
      type: 'text';
      text: string;
    }
  ],
  isError?: boolean;
}
```

Returns an index containing each component and its description.

Part of the [docs toolset](#docs).

### `preview-stories`

Type: `TK`

Returns story previews rendered in the agent's chat interface, if the agent supports [MCP Apps](https://modelcontextprotocol.io/extensions/apps/overview#client-support). Otherwise, it returns links to relevant stories in Storybook.

Part of the [dev toolset](#dev).

### `run-story-tests`

Type: `TK`

If the Storybook has the [Vitest addon](../../writing-tests/integrations/vitest-addon/index.mdx) set up, it runs the tests for specific stories and returns the results, including any accessibility issues if you have [accessibility testing](../../writing-tests/accessibility-testing.mdx) set up. Also instructs the agent to interpret the results and resolve any issues found.

Part of the [testing toolset](#test).
