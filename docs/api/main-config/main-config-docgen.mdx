Configures how Storybook handles automatic documentation generation for component properties, events, and slots.

```ts
// .storybook/main.ts

import type { StorybookConfig } from '@storybook/your-framework';

const config: StorybookConfig = {
  framework: '@storybook/your-framework',
  stories: ['../src/**/*.stories.@(js|jsx|ts|tsx|mdx)'],
  docgen: false, // ðŸ‘ˆ Disable docgen
};

export default config;
```

## `docgen`

Type: `boolean`

Default: `undefined` (docgen enabled)

Enables or disables automatic documentation generation for component properties, events, and slots. When disabled, Storybook will skip the docgen processing step during build, which can improve build performance.

### Disable docgen

```ts
// .storybook/main.ts

import type { StorybookConfig } from '@storybook/vue3-vite'; // or '@storybook/svelte-vite'

const config: StorybookConfig = {
  framework: '@storybook/vue3-vite',
  stories: ['../src/**/*.stories.@(js|jsx|ts|tsx|mdx)'],
  docgen: false,
};

export default config;
```

### When to disable docgen

You might want to disable docgen in the following scenarios:

- **Performance**: Docgen processing can be slow for large projects with many components
- **Not using docs**: If you're not using Storybook's auto-generated documentation features
- **Custom documentation**: If you prefer to manually document your components
- **Build optimization**: To reduce build time in CI/CD pipelines

### Supported frameworks

The `docgen` configuration currently affects the following frameworks:

- `@storybook/svelte-vite`
- `@storybook/vue3-vite`

<Callout variant="info">

React frameworks already have their own docgen configuration via the [`typescript.reactDocgen`](./main-config-typescript.mdx#reactdocgen) option and are not affected by this setting.

</Callout>

### Impact of disabling docgen

When `docgen` is set to `false`:

- Component properties, events, and slots will not be automatically extracted
- The `__docgen` and `__docgenInfo` properties will not be added to components
- Auto-generated controls and documentation will not be available
- Manual documentation and controls can still be used via `argTypes` and `parameters`

### Backward compatibility

- `docgen: undefined` (default) - docgen is enabled
- `docgen: true` - docgen is enabled  
- `docgen: false` - docgen is disabled

This ensures that existing projects continue to work without any changes.
