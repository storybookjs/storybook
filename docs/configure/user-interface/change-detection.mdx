During development, Storybook monitors your git working tree and the builder's module graph to identify which stories are affected by your uncommitted changes. Status icons appear next to stories and components in the sidebar so you can focus your review and testing on what has actually changed.

## Requirements and availability

Change detection is a **development-only** feature. It is active when running `storybook dev` and is not available in static builds (`storybook build`).

The following are required for change detection to work:

- **A git repository.** Storybook uses `git diff` to detect changed files. Change detection is unavailable outside a git repository.
- **A builder that exposes a module graph** (e.g., Vite). Storybook traces changed files through the module import graph to identify all affected stories. Builders that do not expose a module graph will cause change detection to be unavailable.

Change detection starts in one of three readiness states:

| State | Meaning |
| --- | --- |
| **ready** | Change detection is operational and scanning for changes |
| **unavailable** | Change detection cannot run; a reason is provided (e.g., feature disabled, builder does not support module graph, not a git repository) |
| **error** | An unexpected error occurred during initialization or scanning |

If change detection icons never appear in your sidebar, check that both requirements above are met and that the feature is enabled (see [Configuration](#configuration) below).

## Status indicators

Three status values are used by change detection, displayed as accent-colored icons in the sidebar:

| Icon | Status | Meaning |
| --- | --- | --- |
| ✦ Sparkle | **new** | The story file is untracked or newly added in git |
| ● Filled circle | **modified** | A file imported (directly or transitively) by this story was changed |
| ◉ Circle with arcs | **affected** | A story is affected by changes in its dependencies. Difference to **modified**: **affected** indicates indirect changes, while **modified** indicates direct changes |

All three icons use the accent color from your Storybook theme.

The priority ordering when multiple statuses apply is: **new** > **modified** > **affected**.

## Sidebar display

Change detection statuses are displayed alongside test statuses in the sidebar using a dual-slot design.

### Story leaf nodes

Story leaf nodes show a single combined status icon. The `modified` and `affected` statuses are not shown directly on story leaves — they bubble up to the parent component or group node instead. The exception is `new`: a story leaf that is newly added in git shows the sparkle icon directly.

For all other status types (e.g., test results), story leaves show the most critical status across all status types.

## Filtering

You can filter the sidebar to show only stories with a specific change detection status using the filter panel. This lets you quickly isolate new, modified, or affected stories when reviewing a change.

## How it works

When you run `storybook dev`, Storybook subscribes to the builder's module graph via an `onModuleGraphChange` hook. A git diff provider watches your working tree for changes and runs `git diff` to identify modified or new files. Storybook then traces each changed file through the module import graph (by following `import` statements) to find all story files that depend on it — directly or transitively. Stories at import distance 0 are marked **modified**; those at greater distance are marked **affected**. New untracked files are marked **new**.
